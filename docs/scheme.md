## 一、 项目执行摘要 (Executive Summary)

本项目旨在打造一款面向6-15岁K12学生的**游戏化认知能力测评与AI个性化成长建议平台**。产品以“发现学习潜能、提供科学干预”为核心，通过舒尔特方格等标准化认知测试工具，结合多模态AI大模型分析，为家长和教育者提供孩子底层认知能力（注意力、记忆力、逻辑思维等）的科学评估与发展路径规划。

项目采用**Web端（学生测评）+ 多端（家长/教师管理）** 的轻量化架构，以极低的获客门槛实现快速裂变，同时通过WebGL引擎与高精度数据采集技术保障测评的科学性与严谨性。

## 二、 产品定位与商业逻辑

### 2.1 产品定位与核心价值
*   **科学测评**：基于认知心理学标准化工具，用客观行为数据替代主观问卷。
*   **趣味互动**：游戏化设计降低测评焦虑，提升儿童参与意愿。
*   **AI赋能**：大模型驱动的多维度分析，将冰冷的认知指标转化为可执行的学习策略。
*   **隐私合规**：严格遵循未成年人保护法规，数据安全可信赖。

### 2.2 竞品分析与差异化破局点
目前市场存在脑力训练App（重游戏轻评估）、传统量表平台（枯燥且主观）、AI学习机（重学科结果轻底层能力）三类竞品。本产品的核心差异化策略如下：

1.  **从“测智商”降维到“测认知”，从“给分数”升维到“给策略”**：利用AI将认知指标翻译成具体学习场景建议（如：将“视觉广度低”转化为“阅读漏字跳行的干预方案”）。
2.  **构建“客观行为数据”护城河**：通过前端高精度采集微行为特征（轨迹抖动、犹豫停顿），降维打击传统主观问卷。
3.  **首创“认知-学业”映射图谱**：建立底层认知能力与表层学科表现的映射关系，实现“测评-诊断-干预”的商业闭环。

### 2.3 商业模式
*   **C端（家长付费）**：基础测评免费；专业版订阅（¥199/年，含全模块、深度报告、季度追踪）；单次深度报告（¥29/次）。
*   **B端（机构合作）**：学校/教培机构按学生数采购，提供班级学情摸底与因材施教数据支持。

## 三、 核心功能与业务设计

### 3.1 测评工具库（插件化设计）
*   **MVP阶段**：舒尔特方格（视觉注意力）、数字广度（工作记忆）。
*   **扩展阶段**：图形推理（逻辑思维）、快速命名（语言处理）、听觉注意力等。
*   **设计要求**：卡通化UI、适龄难度分级、单次测评≤15分钟防沉迷。

### 3.2 AI评估引擎（核心壁垒）
*   **多模态分析**：结合行为日志（反应时、正确率、操作轨迹）与用户基础信息。
*   **能力画像**：生成雷达图（注意力/记忆力/逻辑/语言/空间/执行功能）。
*   **个性化建议**：基于RAG（检索增强生成）架构，输出训练计划、学习策略及风险预警。

### 3.3 用户信息与档案管理
*   **基础档案**：年龄、年级、兴趣偏好、特殊需求备注（需单独授权）。
*   **成长追踪**：月度/季度能力变化趋势图，可视化呈现干预效果。

## 四、 技术架构与实现方案

### 4.1 架构设计核心原则
1.  **前端游戏化与高精度采集**：Web端采用WebGL保障性能，使用高精度API保障计时准确性。
2.  **后端插件化与高并发**：测评应用与核心业务解耦，新增测试仅需“配置+前端组件”。
3.  **AI原生与RAG架构**：杜绝大模型“幻觉”，AI建议必须基于专业教育学/心理学知识库生成。
4.  **合规前置**：底层实现未成年人数据物理/逻辑隔离与自动脱敏。

### 4.3 核心技术栈选型
| 模块 | 推荐技术栈 | 选型理由 |
| :--- | :--- | :--- |
| **学生端 (Web)** | **Vue 3 + TS + Vite + PixiJS** | Vue3保障开发效率；PixiJS(WebGL)保障复杂测评游戏的60fps流畅渲染。 |
| **家长/教师端** | **Vue 3 + Element Plus / Vant** | 适合复杂图表展示、报告排版和后台管理。 |
| **后端 (业务)** | **Java (Spring Boot 3) / Go** | Java处理复杂业务逻辑，Go处理高并发游戏结果上报网关。 |
| **后端 (AI)** | **Python (FastAPI)** | 方便对接LangChain、大模型SDK和向量数据库。 |
| **数据库** | **MySQL + MongoDB + ClickHouse** | MySQL存核心业务，MongoDB存非结构化行为轨迹，ClickHouse做常模OLAP分析。 |
| **AI 基础设施**| **LangChain + Milvus + 备案大模型API**| 构建RAG应用，Milvus存储教育知识库向量。 |

### 4.4 Web端核心技术难点与解决方案
在Web端做严肃认知测评，必须解决以下痛点：

1.  **时间精度与事件延迟**：
    *   *方案*：全局废弃 `Date.now()`，统一使用微秒级精度的 **`performance.now()`**；废弃 `click`，全面使用 **`PointerEvent`** 消除移动端延迟并兼容多端。
2.  **Web端防作弊与数据真实性**：
    *   *方案*：前端代码深度混淆；不仅上报结果，必须上报**过程轨迹**（坐标、按压时长）；后端风控引擎进行**生理极限校验**（如反应时<150ms判定作弊）和轨迹机器学习识别。
3.  **弱网环境与断线重连**：
    *   *方案*：测评关键状态实时写入 `IndexedDB`；网络恢复后断点续传，意外中断支持恢复上次进度。

## 五、 数据安全与合规策略

合规是K12与AI交叉领域的生命线。必须严格执行以下检查清单：

### 5.1 核心合规检查清单
| 检查维度 | 核心要求 | 落地方案 |
| :--- | :--- | :--- |
| **未成年人保护** | 必须提供“未成年人模式”；限制使用时长（如单日≤40分钟，夜间禁用）。 | 强制引导开启青少年模式；增加夜间强制下线及超时家长密码解锁机制。 |
| **隐私与数据 (PIPL)**| 收集不满14周岁信息需监护人**明示同意**；遵循最小必要原则。 | 注册时强制勾选《儿童隐私规则》并验证家长身份；动态申请摄像头/麦克风权限。 |
| **AI大模型合规** | 调用已通过网信办备案的大模型；AI生成内容需显著标识；防幻觉。 | 选用文心/通义等已备案模型；报告添加“AI辅助生成”水印；设置医疗诊断红线词拦截。 |
| **宣传与商业化** | 禁用“天赋”、“最强大脑”等绝对化/伪科学用语；避免贩卖教育焦虑。 | 对外统一使用“认知能力评估”、“学习潜能诊断”；法务前置审核所有PR与落地页。 |

### 5.2 技术保障措施
*   **数据脱敏网关**：业务调用大模型时，必须经过脱敏网关，将真实姓名、学校替换为匿名标识。
*   **加密与隔离**：敏感字段AES-256加密；未成年人数据独立分区存储；提供一键注销与数据彻底删除功能。

## 六、 项目实施与演进规划

### 6.1 架构与业务演进路线
*   **Phase 1: MVP验证期 (第1-2个月)**
    *   *架构*：模块化单体架构，MySQL+Redis，直接调用大模型API。
    *   *业务*：上线舒尔特方格等2个核心测评，跑通“测评-AI简版报告”闭环，验证C端付费意愿。
*   **Phase 2: 能力扩展期 (第3-5个月)**
    *   *架构*：拆分AI与测评微服务，引入Kafka+ClickHouse，上线RAG知识库架构。
    *   *业务*：新增3-4个测评，AI报告达到专家级水平，接入教育内容资源库，准备公测。
*   **Phase 3: 规模化运营期 (第6个月及以后)**
    *   *架构*：全面云原生(K8s)，开放API网关支持第三方插件接入。
    *   *业务*：全渠道上线，拓展B端学校/机构合作，启动多语言与海外合规调研。

### 6.2 团队配置建议 (MVP阶段)
*   **产品/项目经理** (1人)：需求管理、合规对接、用户研究。
*   **前端开发** (2人)：Web端(学生/家长)开发、PixiJS游戏动效实现。
*   **后端开发** (2人)：业务逻辑、数据管道、安全加固。
*   **AI/算法工程师** (1人)：特征工程、Prompt调试、RAG知识库构建。
*   **教育/心理专家** (1人，顾问)：测评科学性审核、内容把关。

### 6.3 开发启动“第一步”行动指南
技术团队在写下第一行代码前，需完成以下基建：
1.  **制定《Web端高精度埋点与上报协议》**：统一 `performance.now()` 时间戳、PointerEvent坐标、设备DPI等数据格式。
2.  **搭建 WebGL 游戏引擎脚手架**：引入 PixiJS，封装 `<PixiCanvas>` 组件，实现帧率监控与自适应缩放。
3.  **大模型 API 选型与 Prompt 调试**：用模拟数据测试各家大模型生成教育建议的效果与成本，确定底层模型。
4.  **确立前端安全策略**：配置 Vite 代码混淆插件，设计前后端通信的动态签名防重放机制。
