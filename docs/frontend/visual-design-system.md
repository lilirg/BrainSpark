## 设计愿景与原则

### 核心设计愿景

让每个孩子都能在快乐中探索自我，让家长都能在数据中获得信心。

### 设计原则

| 原则 | 说明 |
|------|------|
| **儿童优先** | 学生端以适龄化为核心，降低认知门槛，提升参与感 |
| **信任至上** | 报告端以专业性为核心，信息层次清晰，用数据说话 |
| **一致统一** | 多端共享品牌基因，识别度高但不单调 |
| **包容无障碍** | 遵循 WCAG 2.1 AA 标准，色盲友好、字号可调 |
| **轻交互优先** | 学生端与家长端最小化操作步骤，避免繁杂交互干扰核心任务 |

### 无障碍色彩检查

- 所有文字与背景对比度 ≥ 4.5:1（WCAG AA）
- 禁止仅用颜色传达信息，配合图标/文字
- 提供色盲模式切换，通过不同图案区分（圆点/三角/条纹）

### 字号体系（学生端 - 适龄分级）

| 年龄段 | 字号规范 | 应用说明 |
|--------|----------|----------|
| 6-8 岁 | `48px` 引导标题 / `28px` 方格数字 / `22px` 说明文字 / `18px` 按钮 | 大按钮大字设计，适合触控 |
| 9-11 岁 | `40px` 引导标题 / `24px` 方格数字 / `20px` 说明文字 / `16px` 按钮 | 平衡游戏感与信息量 |
| 12+ 岁 | `32px` 引导标题 / `20px` 方格数字 / `18px` 说明文字 / `15px` 按钮 | 接近成人风格 |

### 字号体系（家长端 / 教师端 - Element Plus 标准）

| 层级 | 字号 | 行高 | 字重 | 使用场景 |
|------|------|------|------|----------|
| H1 | `28px` / `1.75rem` | `1.4` | `600` | 页面主标题 |
| H2 | `22px` / `1.375rem` | `1.4` | `600` | 板块标题 |
| H3 | `18px` / `1.125rem` | `1.5` | `500` | 子标题 |
| Body / Default | `14px` / `0.875rem` | `1.57` | `400` | 正文文本 |
| Small | `12px` / `0.75rem` | `1.5` | `400` | 辅助说明、时间戳 |
| Button | `14px` / `0.875rem` | `1` | `400` | 按钮文本 |
| Caption | `12px` / `0.75rem` | `1.5` | `400` | 说明文字、备注 |

### 字重体系

| 字重 | 值 | 场景 |
|------|-----|------|
| Regular | `400` | 正文、标签 |
| Medium | `500` | 次级标题、Tab 标签 |
| SemiBold | `600` | 标题 H1-H3、重要按钮 |
| Bold | `700` | 大号标题、突出数字 |

### 游戏反馈动画

| 交互事件 | 动画类型 | 持续时长 | 说明 |
|----------|----------|----------|------|
| 正确点击 | 缩放弹跳 + 粒子发散 | 0.4s | 数字方格短暂放大后收缩，青色粒子向外飞出 |
| 错误点击 | 震动抖动（红色） | 0.3s | 方格左右快速摆动 |
| 连续正确 | 光环闪烁（金色） | 0.6s | 连续 5 次正确触发荣耀光环 |
| 任务完成 | 礼花 + 星星下落 | 1.5s | 满屏彩色粒子特效 |
| 获得奖励 | 弹出 + 闪烁旋转 | 0.8s | 星星/徽章从顶部弹出 |

### 进度与计时器

| 组件 | 外观 | 颜色渐变 |
|------|------|----------|
| 时间进度条 | 圆角长条，从左向右填充 | `#00D2FF` → `#FF6B35`（剩余 <20% 变红） |
| 分数计数器 | 滚动数字动画 | 大号字体 + 金色光晕 |
| 连续正确指示器 | 星星/火焰图标序列 | 依次点亮，最新亮起 |

## 轻交互规范

**设计核心：学生端与家长的交互操作最小化，避免繁杂步骤干扰核心任务**

#### 学生端轻交互（游戏期间）

| 交互层级 | 操作数 | 设计意图 |
|----------|--------|----------|
| 登录 | 1-2 步 | 学生码/账号密码一键登录，无额外表单 |
| 任务选择 | 1 步 | 首页直接展示「今日推荐」大按钮，点击即玩 |
| 游戏准备 | 1 步 | 主题以卡片缩略图展示，点击即切换无弹窗 |
| 正式测评 | 0 步 | 游戏界面仅展示方格与计时器，无额外菜单 |
| 结果展示 | 0 步 | 完成后自动弹出动画奖励，无需手动点击下一步 |

**游戏内防干扰规范：**
```
┌─────────────────────────────────────┐
│              顶部状态条               │
│  [计时器]          [音效开关]        │  ← 仅 2 个常驻按钮
├─────────────────────────────────────┤
│                                     │
│    ┌───────────────────┐           │
│    │   游戏画布区域      │           │
│    │   (100% 自适应)     │           │
│    └───────────────────┘           │
│                                     │
│    ↑ 无任何底部导航 ↑                │
│    ↑ 无任何侧边栏 ↑                  │
└─────────────────────────────────────┘
```

| 规则 | 约束 | 说明 |
|------|------|------|
| 无弹窗 | ❌ 游戏期间禁止 | 弹窗打断沉浸感，信息通过 UI 层叠加展示 |
| 无多层菜单 | ❌ 无二级菜单 | 所有操作在一层完成，状态即时切换 |
| 无复杂输入 | ❌ 键盘/文字输入 | 仅支持点击/触摸，6-8 岁完全触控可用 |
| 最小化按钮 | ✅ 最多 4 个 | 开始(返回)、重置、音效、暂停 |
| 自动保存 | ✅ 强制实现 | 关闭浏览器后回到同页可继续，无需手动保存 |

---

#### 家长端轻交互（报告查看）

**核心目标：所见即所得，关键操作不超过三步**

| 用户目标 | 最大步骤数 | 交互设计 |
|----------|-----------|----------|
| 打开最新报告 | ≤ 1 步 | 首页直接展示最新报告入口，无需层层进入 |
| 对比历史数据 | ≤ 2 步 | 报告详情页内嵌「对比」标签，点击即切换 |
| 下载报告 PDF | ≤ 1 步 | 固定位置的「下载」图标，点击即触发 |
| 分享报告链接 | ≤ 1 步 | 生成链接/二维码一键展示，无需跳转设置页 |

**家长端防干扰规范：**
| 规则 | 约束 | 设计意图 |
|------|------|----------|
| 无冗余表单 | ❌ 非必填场景不展示输入框 | 减少无效点击 |
| 无二次确认 | ❌ 下载/分享/对比无需确认框 | 降低操作摩擦 |
| 无全局 Banner | ❌ 通知以 Toast/小红点替代 | 避免视觉噪音 |
| 无多端切换提示 | ❌ 不主动弹出「请在手机端查看」 | 保持当前场景流畅 |
| 渐进式披露 | ✅ 高级选项默认折叠 | 减少界面复杂度，仅核心信息优先展示 |

**设计核心：学生端与家长的交互操作最小化，避免繁杂步骤干扰核心任务**

#### 学生端轻交互（游戏期间）

| 交互层级 | 操作数 | 设计意图 |
|----------|--------|----------|
| 登录 | 1-2 步 | 学生码/账号密码一键登录，无额外表单 |
| 任务选择 | 1 步 | 首页直接展示「今日推荐」大按钮，点击即玩 |
| 游戏准备 | 1 步 | 主题以卡片缩略图展示，点击即切换无弹窗 |
| 正式测评 | 0 步 | 游戏界面仅展示方格与计时器，无额外菜单 |
| 结果展示 | 0 步 | 完成后自动弹出动画奖励，无需手动点击下一步 |

**游戏内防干扰规范：**
```
┌─────────────────────────────────────┐
│              顶部状态条               │
│  [计时器]          [音效开关]        │  ← 仅 2 个常驻按钮
├─────────────────────────────────────┤
│                                     │
│    ┌───────────────────┐           │
│    │   游戏画布区域      │           │
│    │   (100% 自适应)     │           │
│    └───────────────────┘           │
│                                     │
│    ↑ 无任何底部导航 ↑                │
│    ↑ 无任何侧边栏 ↑                  │
└─────────────────────────────────────┘
```

| 规则 | 约束 | 说明 |
|------|------|------|
| 无弹窗 | ❌ 游戏期间禁止 | 弹窗打断沉浸感，信息通过 UI 层叠加展示 |
| 无多层菜单 | ❌ 无二级菜单 | 所有操作在一层完成，状态即时切换 |
| 无复杂输入 | ❌ 键盘/文字输入 | 仅支持点击/触摸，6-8 岁完全触控可用 |
| 最小化按钮 | ✅ 最多 4 个 | 开始(返回)、重置、音效、暂停 |
| 自动保存 | ✅ 强制实现 | 关闭浏览器后回到同页可继续，无需手动保存 |

#### 家长端轻交互（报告查看）

**核心目标：所见即所得，关键操作不超过三步**

| 用户目标 | 最大步骤数 | 交互设计 |
|----------|-----------|----------|
| 打开最新报告 | ≤ 1 步 | 首页直接展示最新报告入口，无需层层进入 |
| 对比历史数据 | ≤ 2 步 | 报告详情页内嵌「对比」标签，点击即切换 |
| 下载报告 PDF | ≤ 1 步 | 固定位置的「下载」图标，点击即触发 |
| 分享报告链接 | ≤ 1 步 | 生成链接/二维码一键展示，无需跳转设置页 |

**家长端防干扰规范：**
| 规则 | 约束 | 设计意图 |
|------|------|----------|
| 无冗余表单 | ❌ 非必填场景不展示输入框 | 减少无效点击 |
| 无二次确认 | ❌ 下载/分享/对比无需确认框 | 降低操作摩擦 |
| 无全局 Banner | ❌ 通知以 Toast/小红点替代 | 避免视觉噪音 |
| 无多端切换提示 | ❌ 不主动弹出「请在手机端查看」 | 保持当前场景流畅 |

## 交互体验规范

### 响应原则

| 交互类型 | 反馈延迟 | 交互方式 | 说明 |
|----------|----------|----------|------|
| 即时操作（点击/切换） | < 100ms | PointerEvent | 零延迟响应 |
| 动画反馈 | 200-400ms | CSS Transition | 平滑过渡 |
| 网络请求 | < 2s | 加载动画 | 避免用户等待焦虑 |
| 超时操作 | > 2s | Skeleton / 进度指示 | 保持进度可见性 |
| 结果呈现 | 200ms 延迟 | 渐入动画 | 增强完成感 |

### 加载状态

| 场景 | 加载形态 | 动画说明 |
|------|----------|----------|
| 页面加载 | Skeleton 骨架屏 | 灰色块渐现渐隐，模拟内容布局 |
| 局部加载 | Spinner 圆形旋转 | 360° 旋转，配合进度百分比 |
| 测评等待 | 进度条 + 鼓励动画 | 百分比进度，配合星星/动画 |
| 空状态 | Illustration + 引导按钮 | 插画占位，提示下一步操作 |

### 提交与反馈

| 操作类型 | 二次确认 | 成功反馈 | 失败反馈 |
|----------|----------|----------|----------|
| 删除账号 | ⚠️ 强确认 | — | 错误 Toast |
| 重置密码 | ⚠️ 强确认 | 成功 Toast | 错误 Toast |
| 发送消息 | 免确认 | 成功 Toast | 错误 Toast |
| 提交测评 | ⚠️ 可撤销 | 动画+跳转 | 保存本地 |

### 无障碍交互

| 需求 | 交互支持 |
|------|----------|
| 键盘导航 | 所有可交互元素 `tabindex` + `:focus` 样式 |
| 屏幕阅读器 | 语义 HTML + `aria-label` / `role` |
| 色盲模式 | 颜色 + 图标 + 文字 三重确认 |
| 减少动画偏好 | 遵循 `prefers-reduced-motion` 媒体查询 |
